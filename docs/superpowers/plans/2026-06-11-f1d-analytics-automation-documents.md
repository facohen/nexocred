# F1d — Tesorería, La Torre, Workflows, Documentos Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish backend executive visibility, automation and document generation: M10 (tesorería — posición, cashflow, DCF, rotación, aportes/retiros), M11 (La Torre — KPIs from persisted snapshots + live alerts/tasks), the snapshot_cartera generation job (admin/on-demand + scheduled), the §7.2 workflow engine (cobranza/novación/CRM families, internal tasks/incidents/alerts only — no WhatsApp), and M13 (documentos — real PDF + SHA-256 hash + transactional per-type numbering + local-filesystem storage adapter + annulment).

**Architecture:** Jobs are **pure/transactional service functions** (testable directly, no Redis), each wrapped by a thin Celery task and exposed via an admin on-demand trigger. La Torre endpoints render from the persisted `snapshot_cartera` (generated by the snapshot job) plus live alert/task data — never hardcoded demo values. The workflow engine evaluates `workflow_regla` rows against trigger contexts and produces only internal effects (`tarea`/`incidente`/`alerta`/admin escalation), idempotent per (regla, trigger-context). Documents generate a real PDF, hash its bytes (SHA-256), assign a transactional per-type correlative `numero`, and persist via a storage adapter (local filesystem for the POC, S3-ready interface).

**Tech Stack:** Python 3.12, FastAPI, SQLAlchemy 2 async, Alembic (migration `0004`), Celery (thin tasks), Redis (prod broker only), `reportlab` or `fpdf2` for PDF, hashlib, pytest, pytest-asyncio. Run in the `nexocred` conda env; Postgres 18 via docker compose. Reuses `nexocred_core`, `app.deps`, `app.locking`, `app.idempotencia`, `app.auditoria`, `app.m04_caja.servicio`, F1c risk `metricas.py`.
