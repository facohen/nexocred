# Frontend Foundation + F1a/F1b Screens Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the NexoCred web app shell (Vite + React + TypeScript + Tailwind + shadcn/ui + TanStack Router/Query/Table) with auth/RBAC navigation, a command palette, a typed API client generated from the frozen OpenAPI contracts, and the F1a/F1b screens: personas, BCRA, catálogo/matrices/simuladores, solicitudes, préstamos, pagos, corrección, caja, novaciones.

**Architecture:** The frontend lives entirely under `frontend/` and is decoupled from the running backend: it talks to a typed client generated from `docs/openapi/{f1a,f1b}.json`, and during development/tests uses **MSW (Mock Service Worker)** handlers seeded with contract-shaped fixtures. Money is always handled as strings (never `float`/`Number` arithmetic on currency) and rendered with `tabular-nums`. Each feature is a folder (`features/<name>/`) with its route, queries, components and tests. Verification is **Vitest + React Testing Library** + `tsc` typecheck + `vite build` — no browser/Postgres dependency.

**Tech Stack:** Node 20, Vite 5, React 18, TypeScript 5, Tailwind CSS, shadcn/ui (Radix), TanStack Router v1, TanStack Query v5, TanStack Table v8, React Hook Form + Zod, `cmdk`, MSW, Vitest, @testing-library/react, `openapi-typescript` (types) + a thin fetch client. Tremor/Recharts deferred to Stage 7 (La Torre).
